## 为什么需要这个工具包？

AI for Science 论文的发表面临独特挑战：

- **审稿人跨领域**：化学/材料专家审ML方法，ML专家审化学发现——两边都要过关
- **可视化标准模糊**：生化环材期刊的图表规范常被算法研究者忽略
- **写作范式特殊**：Nature子刊要"化学洞见优先"而非"算法性能优先"
- **投稿检查分散**：引用真假、数值一致性、交叉引用、AI痕迹检测——每个都要单独工具

本工具包将上述需求整合为 **6个独立模块**，每个模块可单独使用，也可串联成完整pipeline。

---

## 快速开始

```bash
# 克隆后安装依赖
pip install -r requirements.txt

# 按需使用各模块，例如：
# 模块2：设置可视化风格
python module_02_visualization/scripts/style_presets.py
```

## 模块总览

| 模块 | 名称 | 解决什么问题 | 壁垒 |
|------|------|-------------|------|
| **M1** | Nature级写作引擎 | 正文结构、句子节奏、AI痕迹检测、claim强度控制 | 模仿Nature子刊的隐性写作规范 |
| **M2** | 可视化规范与工具 | 字体层级、色盲友好配色、DPI标准、期刊尺寸 | 100+期刊的图表要求预配置 |
| **M3** | 图表验证与交叉引用 | 图-文-数值一致性、全文ref/cite校验 | 程序化验证，不依赖人眼 |
| **M4** | 投稿前全流程Pipeline | 致命错误扫描、编辑速览模拟、数据矛盾检测 | 按期刊/审稿人Persona分级防御 |
| **M5** | 文献管理与引用审查 | DOI→BibTeX、引用真实性验证、批量清洗 | 针对LLM幻觉引用高发区防御 |
| **M6** | LaTeX工具链 | 编译诊断、Unicode清洗、安全批量替换 | 5+年交叉编译实战经验沉淀 |

## 适用人群

- **生化环材博士生**：投Nature子刊/ACS Catalysis/J Catal等，需要写作+图表双保障
- **AI for Science交叉研究者**：计算方法+实验验证混合论文
- **课题组负责人**：建立统一的论文写作和图表规范
- **投稿前最后检查**：快速扫出致命错误

---

## 模块依赖关系

```
M1 写作引擎 ← M5 文献（引用格式）
M2 可视化 → M3 验证（图-文一致）
M4 投稿pipeline → 依赖 M1+M2+M3+M5 的输出
M6 LaTeX工具链 → 独立使用
```

每个模块可独立使用，不强制全栈。
