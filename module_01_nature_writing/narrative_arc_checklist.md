# 叙事弧写作检查清单（Narrative Arc）

> **好的论文像一部三幕剧——悬念、递进、收束。差的论文像实验记录——数据堆砌、没有节奏。**
> 这个清单帮你把论文从"数据报告"升级为"有故事感的科学叙事"。

---

## 一、三幕科学叙事模型

经典科学论文的结构对应三幕剧（Freytag's Pyramid + Hourglass Model）：

```
第一幕：建立（Introduction）
   │  钩子 → 缺口 → 方法预告
   ▼
第二幕：对抗（Results）
   │  核心证据 → 验证 → 机制
   ▼
第三幕：解决（Discussion + Conclusion）
   │  意义 → 边界 → 打开
```

### 信息释放配比

| 幕 | 占全文信息量 | 节奏 |
|----|------------|------|
| Introduction | 20-25% | 钩子(8%) → 缺口(10%) → 预告(5%) |
| Results | 45-55% | 核心发现(20%) → 验证(15%) → 机制/扩展(10-15%) |
| Discussion + Conclusion | 25-30% | 意义(12%) → 边界(8%) → 展望(8%) |

**常见错误**：
- ❌ Introduction 扔了40%的信息，Results 成了重复
- ❌ Results 只报数据不解释，Discussion 才第一次说意义
- ✅ Introduction 吊胃口 → Results 给数据+立即解释 → Discussion 升华

## 二、最强证据的位置

论文最强的那个发现（审稿人最"哇塞"的点），**不要在前30%就拿出来**。

```
正确的节奏：
Abstract:  "揭示了一种新机制"（吊胃口）
Intro:    这个问题前人没解决
Results@40%: 核心证据登场（审稿人已经建立了信任）
Results@60%: 验证 + 机制
```

| ❌ 错误 | ✅ 正确 |
|---------|--------|
| Abstract 直接说最强发现 → 读者还没建立信任 | Abstract 暗示"有发现"但不说是什么 |
| Intro 末段列出所有发现 → Results 没悬念了 | Intro 只说方法预告，结果留在 Results |

---

## 三、证据强度阶梯

论文中的证据不能忽强忽弱，应该像爬楼梯一样递增：

```
L5  独立验证（其他课题组复现、跨体系印证）
L4  机制确认（因果链、实验+计算双证据）
L3  定量对比（统计检验、性能对比）
L2  趋势证据（相关性、排序一致性）
L1  描述性发现（观察、定性比较）
L0  空口断言（没有数据支撑）
```

### 各章节证据强度要求

| 章节 | 最低强度 | 目标强度 |
|------|---------|---------|
| Abstract结论句 | L2 | L2-L3 |
| Introduction引用文献 | L1 | L1-L2 |
| Results开头 | L2 | L3 |
| Results主体 | L3 | L3-L4 |
| Discussion | L2 | L3 |
| Conclusion | L2 | L2-L3 |

### 铁律

- **证据强度只增不减** — Discussion不能比Results弱
- **不能跳级** — L1→L4跳跃让审稿人觉得"凭什么"
- **正确阶梯**：效果好(L1) → 与文献一致(L2) → 统计验证(L3) → 机制确认(L4)

## 四、叙事流汇聚

如果论文包含多条证据线（比如实验+计算、催化A+催化B），它们必须在 Discussion 之前汇聚：

```
流A（核心发现）:  ──┬──────────→
                   │
流B（验证实验）:  ──┼────────→  
                   │
流C（泛化验证）:  ──┴──────→  Discussion
                   ↑
              汇聚点：三条线指向同一结论
```

**规则**：
- 最多3条并行流（再多读者记不住）
- Discussion前必须汇聚
- Discussion中不能出现新流

## 五、段落节奏

### 每小节的"三段体"

每节(paragraph/subsection)的内部节奏：

```
开段（25%信息）→ 铺垫语境，回答"本节要解决什么"
中段（60%信息）→ 数据+图+表，每给一个数字立即解释
末段（15%信息）→ 收束本节发现 + 引出下节问题
```

### 每段必须"收束+打开"

```diff
- ❌ 只收束："以上结果表明MCFF有效。"
- ❌ 只打开："那么MCFF在其他体系上是否也有效呢？"
+ ✅ "MCFF在NH₃-SCR上有效（收束），
+     这种能力是否源自知识融合而非模型复杂度（打开）？
+     下一节通过消融实验解答。"
```

### 节间转场

| ❌ 禁止 | ✅ 推荐 |
|--------|--------|
| "Another important aspect is..." | 问题转场：收束→问问题→下节回答 |
| "The next experiment examined..." | 尺度转场：从原子级→反应器级 |
| 空行+直接给数据 | 对比转场："这与文献一致，但在X体系中尚不清楚" |

## 六、投稿前快速审计

### 协议A：信息释放位置

```
逐段问：最强证据第一次出现在全文哪个位置？
🔴 前30%就出现了 → 太早，读者还没建立信任
🟢 40-60%位置 → 理想
```

### 协议B：证据强度单调性

```
从Abstract到Conclusion逐段检查证据强度：
🔴 Discussion < Results → 证据曲线倒挂，必须修复
🟢 单调递增或持平 → 通过
```

### 协议C：流汇聚

```
🔴 Discussion时还有≥2条独立的流 → 没汇聚，必改
🟢 Discussion前已汇聚 → 通过
```
